# 图书馆管理系统 — 使用文档

## 目录结构

```
library-system/
├── 功能.md                  # 功能规划
├── 使用指南文档.md          # ← 当前文件
├── docker-compose.yml       # 生产环境（默认）
├── docker-compose.dev.yml   # 开发环境
├── docker-compose.prod.yml  # 生产环境
│
├── backend/                 # 后端 Express + TypeScript
│   ├── src/index.ts         #   入口：中间件 + 路由注册
│   ├── src/config/          #   数据库连接 + 类型定义
│   ├── src/middleware/       #   JWT 认证 + 角色权限
│   ├── src/routes/          #   路由层（参数校验 + 响应）
│   └── src/api/             #   数据访问层（SQL 查询）
│
├── frontend/                # 前端 Vue 3 + TypeScript
│   ├── src/api/             #   HTTP 请求封装
│   ├── src/stores/          #   Pinia 状态管理
│   ├── src/router/          #   路由配置 + 守卫
│   ├── src/views/           #   页面组件
│   └── src/config/          #   前端类型定义
│
├── mysql/
│   └── init/                # 数据库初始化脚本
│       ├── init.sql         #   建表
│       └── seed.sql         #   种子数据
│
└── nginx/
    └── conf.d/default.conf  # 反向代理配置
```

## 🖥️ 全新电脑 — Docker 方案（推荐）

### Windows 11

```powershell
# 1. 安装 Docker Desktop
#    从 https://docs.docker.com/desktop/setup/install/windows-install/ 下载
#    安装后打开 Docker Desktop，等待引擎启动

# 2. 进入项目目录
cd C:\path\to\library-system

# 3. 构建前端
cd frontend
npm install
npm run build
cd ..

# 4. 启动全部服务
docker compose up -d

# 5. 打开浏览器访问 http://localhost
```

### Ubuntu

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
# 注销重新登录或执行: newgrp docker

# 2. 进入项目目录
cd /path/to/library-system

# 3. 构建前端
cd frontend && npm install && npm run build && cd ..

# 4. 启动全部
docker compose up -d

# 5. 打开 http://localhost
```

## 🖥️ 全新电脑 — 无 Docker

### Windows 11

```powershell
# 1. 安装 Node.js (https://nodejs.org) v22 LTS
# 2. 安装 MySQL (https://dev.mysql.com/downloads/installer/)
#    设置 root 密码为 123456

# 3. 管理员 PowerShell，初始化数据库
cd C:\path\to\library-system
mysql -uroot -p123456 -e "CREATE DATABASE IF NOT EXISTS library"
mysql -uroot -p123456 library < mysql\init\init.sql
mysql -uroot -p123456 library < mysql\init\seed.sql

# 4. 启动后端（新开终端）
cd backend
npm install
npm run dev

# 5. 启动前端（再开一个终端）
cd frontend
npm install
npm run dev
```

访问 `http://localhost:5173`。

### Ubuntu

```bash
# 1. 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash
sudo apt install -y nodejs

# 2. 安装 MySQL
sudo apt install -y mysql-server
sudo mysql -e "ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '123456'; FLUSH PRIVILEGES;"

# 3. 初始化数据库
mysql -uroot -p123456 -e "CREATE DATABASE IF NOT EXISTS library"
mysql -uroot -p123456 library < mysql/init/init.sql
mysql -uroot -p123456 library < mysql/init/seed.sql

# 4. 启动后端
cd backend && npm install && npm run dev &

# 5. 启动前端
cd frontend && npm install && npm run dev
```

访问 `http://localhost:5173`。

## 🔌 API 接口

所有接口前缀 `/api`，需在 Header 传 `Authorization: Bearer <token>`。

### 认证

| 方法 | 路径      | 说明             |
| ---- | --------- | ---------------- |
| POST | `/login`  | 登录，返回 token |
| POST | `/logout` | 退出，token 失效 |

### 用户

| 方法   | 路径                        | 权限   | 说明                       |
| ------ | --------------------------- | ------ | -------------------------- |
| GET    | `/users`                    | 登录   | 用户列表（分页+搜索）      |
| GET    | `/users/search`             | 登录   | 精简搜索（借书弹窗用）     |
| GET    | `/users/:id`                | 登录   | 用户详情                   |
| GET    | `/users/:id/stats`          | 登录   | 借阅统计                   |
| POST   | `/users`                    | 管理员 | 创建用户                   |
| PUT    | `/users/:id`                | 管理员 | 更新用户                   |
| PUT    | `/users/password`           | 登录   | 修改自己的密码             |
| PUT    | `/users/:id/reset-password` | 管理员 | 重置他人密码               |
| DELETE | `/users/:id`                | 管理员 | 删除用户（有未还书时禁止） |

### 图书

| 方法   | 路径            | 权限   | 说明                           |
| ------ | --------------- | ------ | ------------------------------ |
| GET    | `/books`        | 登录   | 图书列表（分页+搜索+分类筛选） |
| GET    | `/books/search` | 登录   | 精简搜索（借书弹窗用）         |
| GET    | `/books/:id`    | 登录   | 图书详情                       |
| POST   | `/books`        | 管理员 | 创建图书                       |
| PUT    | `/books/:id`    | 管理员 | 更新图书                       |
| DELETE | `/books/:id`    | 管理员 | 删除图书（有未还书时禁止）     |

### 分类

| 方法   | 路径              | 权限   | 说明                     |
| ------ | ----------------- | ------ | ------------------------ |
| GET    | `/categories`     | 登录   | 所有分类                 |
| GET    | `/categories/:id` | 登录   | 分类详情                 |
| POST   | `/categories`     | 管理员 | 创建分类                 |
| PUT    | `/categories/:id` | 管理员 | 更新分类                 |
| DELETE | `/categories/:id` | 管理员 | 删除分类（有图书时禁止） |

### 借阅

| 方法 | 路径                  | 权限   | 说明                               |
| ---- | --------------------- | ------ | ---------------------------------- |
| GET  | `/borrows`            | 管理员 | 全部借阅记录（分页+搜索+状态筛选） |
| GET  | `/borrows/my`         | 登录   | 当前用户的借阅记录                 |
| GET  | `/borrows/:id`        | 登录   | 借阅详情                           |
| POST | `/borrows`            | 管理员 | 借书（扣减可借数）                 |
| PUT  | `/borrows/:id/return` | 管理员 | 还书（恢复可借数）                 |
| PUT  | `/borrows/:id/renew`  | 管理员 | 续借                               |

### 仪表盘

| GET  | `/dashboard` | 统计 + 近7天趋势 + 热门TOP10 + 逾期列表（普通用户仅看自己） |

## 🐳 Dockerfile 说明

后端 `Dockerfile.prod` 使用多阶段构建：

```
阶段 1 (builder)     →     阶段 2 (运行)
FROM node:22-alpine        FROM node:22-alpine
npm ci                     npm ci --production
npm run build              只复制 dist/
→ 输出 dist/               → node dist/index.js
```

最终镜像约 60MB，仅包含编译后的 JS 文件和生产依赖。

手动构建：

```bash
cd backend
docker build -t library-backend -f Dockerfile.prod .
```

## 🔧 常见问题

### 登录提示"用户名或密码错误"

```bash
# 检查用户是否存在
docker exec library-mysql mysql -uroot -p123456 library \
  -e "SELECT id,username,name FROM users"

# 缺少 admin 时重新插入
docker exec -i library-mysql mysql -uroot -p123456 library <<< \
  "INSERT IGNORE INTO users (username, password, name, role)
   VALUES ('admin', '\$2b\$10\$uqqR2sS5q/zhdt5vBty5y.BKzj0WFdVN4iZLlpc9mkfvER8WA0Xz.', '系统管理员', 'admin');"
```

### Nginx 报错 "host not found in upstream"

容器不在同一网络，重启即可：

```bash
docker compose down
docker compose up -d
```

### 端口被占用

修改 `docker-compose.yml` 中 `ports` 映射，例如：

```yaml
ports:
  - "8080:80" # 将 Nginx 80 端口映射到宿主机 8080
  - "3001:3000" # 将后端 3000 端口映射到宿主机 3001
```

## ⚙️ 环境变量

| 变量             | 默认值      | 说明                              |
| ---------------- | ----------- | --------------------------------- |
| `NODE_ENV`       | development | 运行环境                          |
| `PORT`           | 3000        | 后端端口                          |
| `DB_HOST`        | localhost   | 数据库地址（Docker 内用 `mysql`） |
| `DB_PORT`        | 3306        | 数据库端口                        |
| `DB_USER`        | root        | 数据库用户                        |
| `DB_PASSWORD`    | 123456      | 数据库密码                        |
| `DB_NAME`        | library     | 数据库名                          |
| `JWT_SECRET`     | (随机)      | 签名密钥                          |
| `JWT_EXPIRES_IN` | 3d          | Token 过期时间                    |
